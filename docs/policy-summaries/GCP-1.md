# GCP-1 정책 요약

> 이 문서는 공개 프로토타입에서 사용하는 정책 요약입니다. 원본 PRD 전체를 복제하지 않으며, 원본 정책을 독립적으로 수정하지 않습니다.

## 목적

- 병원이 등록한 진료항목을 굿닥과 외부 예약 채널에서 일관되게 운영합니다.
- 채널별 예약 신청은 Connect의 기존 운영 흐름 안에서 함께 관리합니다.

## 현재 프로토타입 반영 기준

- 진료항목 목록에서 굿닥과 카카오톡 예약하기의 노출 상태를 함께 확인합니다. 목록의 채널 심볼은 읽기 전용 현황이며 카카오 노출 설정은 상세에서 변경합니다.
- 카카오 노출 여부와 카카오 전용 정보는 진료항목 상세에서 관리합니다. 카카오 노출을 ON하면 별도 추가 토글 없이 전용 입력 필드를 즉시 표시합니다.
- V1에서는 굿닥에 노출 중인 진료항목만 카카오톡 예약하기에도 노출할 수 있습니다.
- 굿닥 노출을 끄면 카카오 노출 의도와 전용정보는 보존하되 실제 카카오 상품은 노출 보류합니다. 굿닥을 다시 켜면 기존 카카오 노출 의도가 ON인 상품만 외부 반영 성공 후 복원합니다.
- 굿닥과 카카오 예약은 예약 신청 내역에서 채널을 구분해 함께 처리하며, 채널 열은 진료항목과 방문자 사이에 표시합니다.
- 예약 채널은 현재 상품 상태가 아니라 해당 예약이 실제로 유입된 채널을 기준으로 표시합니다.
- 카카오에서 수집된 추가 질문·답변이 있는 경우, 모든 예약 상태의 개별 상세 `요청사항` 하위에 예약 당시 문구와 순서대로 표시합니다. 데이터가 없으면 추가 영역은 표시하지 않습니다.
- 비급여 예약 운영을 중지하면 신규 예약 슬롯 제공을 중단하지만 등록된 진료항목과 기존 예약은 유지합니다.
- 상품별 노출 의도, 현재 예약 가능 여부, 외부 실제 상태와 동기화 상태를 구분합니다. 외부 반영 전에는 `노출 중`으로 표시하지 않습니다.
- 노출 의도와 외부 반영 상태는 내부적으로 구분하고 실패 작업만 안전하게 재처리합니다. 이를 조작하는 상태 선택기나 기술 상태표는 병원 제품 화면에 추가하지 않습니다.
- 상세에서 카카오 노출을 ON 하면 검색 키워드, 이용 방법(최대 2,000자), 방문 안내(최대 1,000자), 취소 유의사항(최대 100자) 입력 필드가 바로 표시됩니다.
- 카카오 상품명·설명·대표 이미지·상세 이미지는 공통 진료항목 정보를 사용하므로 별도 입력 필드를 제공하지 않습니다. 공통 대표 이미지가 없으면 카카오 장소 이미지를 사용합니다.
- V1 병원 화면에는 판매기간, 강제 품절 표시, 상품 노출순서, 예약 부가정보 질문 편집을 제공하지 않습니다. 다만 카카오 예약에서 이미 수집된 추가 질문·답변은 기존 정책대로 예약 상세에 조건부 표시합니다.
- 굿닥 가격 옵션을 `[가격 정보] - 가격 설명` 형식의 카카오 `Price.description`으로 변환하며 최종 문구가 100자를 넘으면 저장을 막습니다. 고정가는 고정 금액, 할인가는 판매가만, 상담 후 결정은 `상담 후 결정`을 사용합니다. 별도의 카카오 가격 안내 미리보기는 제공하지 않습니다. 신규 GCP-1 Product는 가격 옵션 수와 관계없이 항상 `priceDisplayType=SELECT`로 생성하며, 1↔N 변경은 Product를 유지하고 Price만 증감합니다. 활성 Price가 0개면 카카오 노출을 차단합니다.
- 굿닥 저장 성공과 카카오 반영 성공을 분리하고 Product·기술 Item(`옵션`)·Price·Schedule별 상태와 실패 사유는 내부 동기화·관제 정보로 관리합니다.
- 활성 또는 미래 예약이 있는 진료항목은 물리 삭제하지 않고 운영을 중지하며 기존 예약과 외부 매핑을 보존합니다.
- 비급여 예약 전체 OFF는 상품별 노출 의도를 보존한 채 외부 Product·향후 Schedule을 노출 보류로 전환합니다. 다시 ON하면 유효한 상품만 선별해 외부 반영 성공 후 복원합니다.
- 자동 확정과 당일 예약 설정은 외부 예약에도 적용합니다. 카카오 예약에는 굿닥 신규 예약 알림을 발행하지 않습니다.
- 예약에는 당시 상품·가격·채널·자동 확정 설정을 보존하고 외부 상태 반영 실패는 내부 재처리합니다. 외부 ID·적용 버전·알림 발행 결과 같은 진단 정보는 병원 화면에 추가하지 않습니다.

## 외부 예약 흐름

1. 카카오에서 진료항목형 상품, 가격 옵션과 일정을 선택합니다.
2. 카카오에서 예약자·방문자 기본정보와 동의를 입력합니다.
3. 굿닥 예약 웹뷰로 이동해 필요한 추가정보를 입력합니다.
4. 굿닥이 병원 운영 가능 여부와 공통 예약 슬롯을 최종 확인합니다.
5. 조건을 만족하면 통합 예약 원장에 한 번만 생성하고, 마감 또는 오류이면 재선택 안내를 표시합니다.

기술 `Item(옵션)`은 외부 API 구조를 맞추는 객체이며 환자에게 별도 선택 단계로 제공하지 않습니다.

## 검토 중인 화면 노출 조건

> 아래 항목은 최종 PRD에 확정된 조건이며, 프로토타입의 `예정된 내용도 보기`를 켰을 때 확인할 수 있습니다.

- **병원 단위 카카오 연동 여부**는 진료항목 목록의 카카오 채널 정보, 진료항목 상세의 카카오 설정 영역, 예약 신청 내역의 채널 열을 표시할지 결정합니다.
- **개별 진료항목의 카카오 연동 여부**는 위 영역들의 표시 여부가 아니라 해당 상품의 카카오 노출 상태와 전용 입력 필드 상태를 결정합니다.
- 병원이 카카오에 연동되어 있다면 개별 진료항목이 아직 카카오에 노출되지 않았더라도 목록의 카카오 채널 정보와 예약 신청 내역의 채널 열은 유지합니다.
- 병원이 카카오에 연동되지 않았다면 위 세 화면의 카카오 관련 영역을 표시하지 않습니다.
