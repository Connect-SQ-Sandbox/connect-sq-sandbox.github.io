# GCP-1 정책 요약

> 이 문서는 공개 프로토타입에서 사용하는 정책 요약입니다. 원본 PRD 전체를 복제하지 않으며, 원본 정책을 독립적으로 수정하지 않습니다.

## 목적

- 병원이 등록한 진료항목을 굿닥과 외부 예약 채널에서 일관되게 운영합니다.
- 채널별 예약 신청은 Connect의 기존 운영 흐름 안에서 함께 관리합니다.

## 현재 프로토타입 반영 기준

- 진료항목 목록에서 굿닥과 카카오톡 예약하기의 노출 상태를 함께 확인합니다. 목록의 채널 심볼은 읽기 전용 현황이며 카카오 노출 설정은 상세에서 변경합니다.
- 카카오 노출 여부와 카카오 전용 정보는 진료항목 상세에서 관리합니다. 카카오 노출을 ON하면 별도 추가 토글 없이 전용 입력 필드를 즉시 표시합니다.
- V1에서는 굿닥에 노출 중인 진료항목만 카카오톡 예약하기에도 노출할 수 있습니다.
- 굿닥 노출을 끄면 카카오 노출도 함께 끄고, 카카오만 끄는 것은 허용합니다. 굿닥을 다시 켜도 카카오 노출은 자동으로 켜지지 않습니다.
- 굿닥과 카카오 예약은 예약 신청 내역에서 채널을 구분해 함께 처리하며, 채널 열은 진료항목과 방문자 사이에 표시합니다.
- 예약 채널은 현재 상품 상태가 아니라 해당 예약이 실제로 유입된 채널을 기준으로 표시합니다.
- 카카오에서 수집된 추가 질문·답변이 있는 경우, 모든 예약 상태의 개별 상세 `요청사항` 하위에 예약 당시 문구와 순서대로 표시합니다. 데이터가 없으면 추가 영역은 표시하지 않습니다.
- 비급여 예약 운영을 중지하면 신규 예약 슬롯 제공을 중단하지만 등록된 진료항목과 기존 예약은 유지합니다.

## 검토 중인 화면 노출 조건

> 아래 항목은 원본 PRD가 review 상태인 예정 조건이며, 프로토타입의 `예정된 내용도 보기`를 켰을 때 확인할 수 있습니다.

- **병원 단위 카카오 연동 여부**는 진료항목 목록의 카카오 채널 정보, 진료항목 상세의 카카오 설정 영역, 예약 신청 내역의 채널 열을 표시할지 결정합니다.
- **개별 진료항목의 카카오 연동 여부**는 위 영역들의 표시 여부가 아니라 해당 상품의 카카오 노출 상태와 전용 입력 필드 상태를 결정합니다.
- 병원이 카카오에 연동되어 있다면 개별 진료항목이 아직 카카오에 노출되지 않았더라도 목록의 카카오 채널 정보와 예약 신청 내역의 채널 열은 유지합니다.
- 병원이 카카오에 연동되지 않았다면 위 세 화면의 카카오 관련 영역을 표시하지 않습니다.
