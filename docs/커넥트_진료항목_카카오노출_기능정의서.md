# 진료항목 정보 기반 카카오 노출 — 기능정의서

> 대상 프로토타입: `pages/connect/ti-kakao/index.page.tsx`
> 성격: 굿닥 진료항목 정보를 기반으로, 원하면 카카오톡 예약하기 상품으로도 노출하는 UX. **자동 매핑 + 검증**(운영자가 카카오 값을 재입력하지 않고, 굿닥 정보가 카카오 규격으로 자동 반영되며 규격 위반만 검증으로 표시).
> 화면 4개: ① 진료항목 목록 ② 진료항목 폼(등록/상세) ③ 예약 신청 내역 ④ 운영 설정. 카카오 전용 목록·화면은 두지 않고 기존 Connect 흐름에 병합.

---

## 진료항목 목록

기존 진료항목 관리 화면(2단 그리드: 좌 카테고리 / 우 항목 리스트)을 그대로 두고, 항목 행에 **인입 채널 표기**만 추가.

### 헤더

- 타이틀: '진료항목'
- 서브텍스트: '굿닥에 노출되는 우리 병원 진료항목과 가격 정보를 관리할 수 있어요. 카카오톡 예약하기 노출 여부도 여기서 확인해요.'

#### [새 진료항목]

- 기본 값 : 항상 활성화
- 클릭 시 : 진료항목 폼을 신규 등록 모드로 진입
  - 신규 draft의 카테고리(master1/master2)는 현재 선택된 카테고리 기준으로 초기화

### 좌측 카테고리 (master1 nav)

- 항목이 1개 이상 있는 카테고리만 노출 (예: 피부·미용 / 성형·윤곽 / 주사·수액 / 직접 입력 항목)
- 각 카테고리 우측에 소속 항목 수 표기
- 기본 선택: 첫 카테고리(피부·미용)
- 선택 상태: 좌측 파란 바 + 파란 텍스트

##### [카테고리 항목] 클릭

- 클릭 시 : 선택 카테고리 전환 → 우측 항목 리스트가 해당 카테고리로 갱신
- ※ 드래그 핸들은 hover 시 노출되나 정렬 동작은 미구현(시각만)

### 우측 항목 리스트

- 선택 카테고리가 '직접 입력 항목'인 경우: 그룹 없이 항목 행을 평면 나열
- 그 외 표준 카테고리: master2 그룹 단위로 묶어 표시
  - 각 그룹: 그룹명(master2) 헤더 + 소속 항목 행

### 항목 행

구성(좌→우): 드래그 핸들 / 항목명 / 대표가격 + 옵션 수 / 썸네일 / 인입 채널 표기 / 굿닥 노출 토글 / 삭제

#### 항목명·가격 표기

- 항목명: 노출명이 있으면 노출명, 없으면 진료항목명
- 대표가격: 첫 가격 옵션 기준. 고정=금액, 할인=판매가, 상담=‘상담 후 결정’. 옵션이 2개 이상이면 ‘~’ 접미
- 옵션 수: 가격 옵션 개수를 회색 태그로 표기
- 썸네일: 대표 이미지 있으면 이미지, 없으면 기본 플레이스홀더 아이콘

#### 인입 채널 표기 [굿닥 심볼][카카오 심볼]

두 채널 심볼 아이콘을 나란히 노출(텍스트 없음). 상태를 아이콘으로만 구분하며 hover 시 툴팁으로 의미 전달.

- 굿닥 심볼
  - 노출 중(굿닥 노출 ON): 풀컬러
  - 미노출(굿닥 노출 OFF): 회색(흐리게)
  - 툴팁: '굿닥 노출 중' / '굿닥 미노출'
- 카카오 심볼
  - 노출 중(카카오 노출 ON) && 규격 경고 0: 풀컬러(노란 카카오)
  - 노출 중 && 규격 경고 1건 이상: 풀컬러 + 우상단 주황 점
  - 미노출(카카오 노출 OFF): 회색(흐리게)
  - 툴팁: '카카오톡 예약하기 미노출' / '카카오톡 예약하기 노출 중' / '카카오톡 예약하기 노출 중 · 규격 검토 N건'
- ※ 채널 노출 상태는 이 목록에서 편집하지 않음. 카카오 노출 ON/OFF·전용 정보는 진료항목 폼의 ‘카카오톡 예약하기 노출’ 섹션에서 제어

##### [굿닥 노출 토글]

- 기본 값 : 항목의 현재 노출 상태
- 클릭 시 :
  - 해당 항목의 굿닥 노출 상태를 즉시 토글
  - 좌측 굿닥 심볼 색상(컬러↔회색)이 함께 갱신
- ※ 행 클릭(상세 진입)과 분리 처리 — 토글 클릭은 상세로 이동하지 않음

##### [항목 행] 클릭 (토글·삭제 영역 제외)

- 클릭 시 : 해당 진료항목을 폼(상세) 모드로 열기

##### [삭제]

- ※ 프로토타입에서는 hover 시 노출되는 시각 요소만 존재, 동작 미연결

## 진료항목 폼 (등록 / 상세)

좌측 입력 폼 + 우측 굿닥 기준 라이브 미리보기 + 하단 고정 풋터.

### 헤더

- [목록] : 클릭 시 목록으로 복귀
- 타이틀: 신규 진입 시 '진료항목 등록', 기존 항목 진입 시 '진료항목 정보'

### 필수 정보

#### 진료항목명

- 검색/직접입력 인풋
- 입력 값은 우측 미리보기 타이틀에 실시간 반영(노출명이 없을 때)

#### 가격 정보

가격 옵션을 1개 이상 등록. 각 옵션 단위로 정의.

##### 가격 옵션

- 가격명: 최대 50자
- 가격 설명: 최대 100자(선택)
- 가격 타입 드롭다운: '고정 가격' / '할인 가격' / '상담 후 결정'
  - 고정 가격: 금액 입력 1개 + ‘원’
  - 할인 가격: 정상가 → 판매가 2개 입력 + ‘원’
  - 상담 후 결정: 금액 입력 비활성(‘0’ 고정 표시)
- 카카오 노출이 ON인 경우 인라인 검증(해당 옵션에만 표시):
  - 가격명 25자 초과 시: 경고 ‘카카오 상품에선 25자까지 노출돼요 (현재 N자)’
  - 타입이 할인 가격인 경우: 안내 ‘카카오엔 판매가 {금액}만 참고가로 노출돼요.’
- 옵션 삭제: 옵션이 1개일 때는 삭제 불가

##### [가격 옵션 추가]

- 클릭 시 : 빈 가격 옵션 행 1개 추가(기본 타입 ‘고정 가격’)

### 추가 정보

#### 대표 사진

- 미등록 상태: [사진 추가] 버튼
  - 클릭 시 : 대표 이미지 등록(프로토타입은 샘플 이미지 mock) → 미리보기 대표 이미지 갱신
- 등록 상태: 썸네일 + [X]
  - [X] 클릭 시 : 대표 이미지 제거

#### 진료항목 노출명

- 최대 50자, 하단 글자 수 카운터
- 값이 있으면 목록·미리보기에서 항목명 대신 노출명 사용

#### 한 줄 소개

- 최대 50자, 하단 글자 수 카운터
- 미리보기 소개 문구에 실시간 반영

#### 진료항목 키워드

- 입력 후 Enter 시 칩 추가(최대 20개), 하단 개수 카운터
- 칩의 [×] 클릭 시 해당 키워드 삭제
- ※ 중복 키워드·20개 초과 시 추가되지 않음

### 카카오톡 예약하기 노출 (섹션)

#### [카카오에 노출 토글]

- 위치: 섹션 헤더 우측(카카오 심볼 + '카카오톡 예약하기 노출' 라벨 옆)
- 기본 값 : 항목의 현재 카카오 노출 상태(신규 등록 시 OFF)
- 활성화 조건: 카카오톡 예약하기 연동 병원인 경우에만 조작 가능
  - ※ 미연동 병원: 토글 비활성 처리
  - ※ 프로토타입은 연동 병원(항상 조작 가능)으로 시뮬
- ON일 때만 하위 영역(자동 반영 안내 / 검증 결과 / 카카오 전용 정보) 노출

#### 자동 반영 안내

- 카카오 노출 ON 시 상단 안내: '위에 입력한 굿닥 진료항목 정보가 카카오 상품으로 자동 반영돼요. 규격에 안 맞는 부분만 아래에서 확인하세요.'

#### 검증 결과

- 카카오 노출 ON 시, 현재 입력값 기준 규격 위반을 실시간 계산해 리스트로 표시
- 레벨: '경고'(규격상 잘리거나 미지원) / '안내'(정보성)
- 위반 항목이 없으면: '규격에 모두 맞아요. 그대로 노출돼요.'
- 항목별 표기: 레벨 아이콘 + 필드명 + 메시지 (규칙은 아래 ‘카카오 규격 검증 규칙’ 참조)

#### 카카오 전용 정보 (아코디언, 선택)

- [카카오 전용 정보 추가 입력] 토글 버튼
  - 기본 값 : 접힘
  - 클릭 시 : 펼침/접힘 전환
- 펼침 시 입력 항목(카카오톡 예약하기에만 노출, 굿닥 화면에는 미노출):
  - 예약 시 받을 정보(질문 리스트)
    - [질문 추가] 클릭 시 : 빈 질문 행 추가
    - 질문 행: 질문 텍스트 입력 + [선택/필수] 토글(클릭 시 선택↔필수 전환) + [×] 삭제
  - 이용 방법: 텍스트 입력
  - 유의사항: 텍스트 입력
  - 취소 유의사항: 텍스트 입력(최대 100자)

### 우측 미리보기 (굿닥 기준)

- 좌측 폼 입력값을 실시간 반영하는 모바일 프레임
- 노출 요소: 대표 이미지 / 타이틀(노출명 우선, 없으면 진료항목명, 둘 다 없으면 placeholder) / 한 줄 소개 / 키워드 칩 / 가격 목록
  - 가격 목록: 할인 옵션은 정상가 취소선 + 판매가
- 하단 라벨: '굿닥 기준 미리보기'
- ※ 카카오 미리보기는 제공하지 않음(미리보기 영역은 굿닥 기준 전용)

### 하단 풋터

##### [취소]

- 클릭 시 : 저장 없이 목록으로 복귀

##### [저장]

- 클릭 시 :
  - 신규: 목록에 항목 추가 + 토스트 '진료항목을 등록했어요.'
  - 기존: 항목 갱신 + 토스트 '진료항목을 저장했어요.'
  - 저장 후 목록으로 복귀
- ※ 프로토타입은 mock 저장(실제 API 연동 없음)

#### 카카오 상태 표기 (풋터 좌측, 카카오 노출 ON일 때)

- 규격 경고 1건 이상: '카카오 검토 N건 (자동 보정돼 노출)' + 경고 아이콘
- 규격 경고 0건: '카카오 규격 확인됨' + 카카오 심볼

##### [굿닥 노출 토글] (풋터 우측)

- 라벨: 노출 시 '환자들에게 진료항목을 노출합니다.', 미노출 시 '진료항목을 노출하지 않습니다.'
- 클릭 시 : 굿닥 노출 상태 토글

## 카카오 규격 검증 규칙 (참조)

카카오 노출 ON일 때, 현재 입력값으로 실시간 계산. ‘경고’는 검토 카운트에 포함, ‘안내’는 제외.

- 상품명(노출명 우선, 없으면 진료항목명)
  - 50자 초과: 경고 — 50자로 잘려서 노출됨
- 가격명(옵션별)
  - 25자 초과: 경고 — 25자로 잘려서 노출됨
- 할인 가격 옵션 존재
  - 경고 — 정상가·할인율 미지원, 판매가만 참고가로 노출
- 유료(상담 아님) 가격 존재
  - 안내 — 카카오는 결제 없이 참고가로만 노출
- 대표 이미지 없음
  - 안내 — 카카오에선 병원 대표 이미지로 대체

## 프로토타입 범위 / 실서비스 확인 필요

프로토타입에서 mock 처리했거나 정의되지 않아, 실서비스 설계 시 확정이 필요한 항목.

- 저장/취소/삭제·토스트는 mock (실제 저장·삭제 API 미연동)
- 항목/카테고리 드래그 정렬: 핸들 시각만 있고 동작 미구현
- 항목 행의 [삭제]: 시각만, 동작 미연결
- 카카오 연동 병원 여부: 항상 연동으로 시뮬(실제로는 연동 상태에 따라 토글 활성/비활성 분기)
- 카카오 노출 ON→실제 카카오 상품 등록/동기화 시점·주체, OFF→상품 내림 처리: 미정
- 예약 일정(스케줄)·재고 등 카카오 예약 상품 필수 요소는 본 화면 범위 밖(정보 노출 규격 중심). 카카오 실제 예약을 받으려면 스케줄 설정 연계 필요 여부 확정 필요
- 규격 경고가 있는 상태로 노출 시 ‘자동 보정(잘림 등)’ 후 노출한다는 정책의 확정 여부

## 정책 변경 패널

- 화면 우측 `변경사항` 버튼으로 열며 기본값은 닫힘이다.
- 실제 Connect 제품 기능이 아니라 프로토타입 검토를 위한 메타 UI다.
- 기본 목록은 현재 화면에 연결된 변경사항이며 `전체`로 다른 화면 변경도 볼 수 있다.
- 패널 상단의 `예정된 내용도 보기`는 기본 OFF다. ON 시 `draft` 또는 `review` PRD에서 공개 가능한 예정 변경과 화면 미리보기를 함께 노출한다.
- 예정 모드가 켜진 동안 화면 상단에 `예정 화면 미리보기` 안내를 유지해 현재 정책과 혼동되지 않게 한다.
- 각 변경은 원본 `PRD ID`, 변경 전후 요약, 프로토타입 버전을 표시한다.
- 각 변경 셀 하단에는 원본 PRD의 `target_release_at`을 예정 배포일로 표시하며 값이 `null`이면 `미정`으로 표기한다.
- 변경 항목 클릭 시 해당 화면으로 이동하고 `data-policy-id`로 연결된 영역을 강조한다.
- `PRD 요약 보기`는 전체 내부 PRD가 아니라 `docs/policy-summaries/`의 공개 가능한 Markdown 요약을 패널에서 보여준다.
- 원본 PRD가 승인되지 않은 기존 공개 화면은 `현재 기준선`으로 표시하며 `승인 반영`으로 오인시키지 않는다.
- 현재 첫 예정 예시는 GCP-1의 `굿닥 OFF·카카오 ON` 독립 노출안이다. 토글 ON 시 목록에서 굿닥 미노출·카카오 노출 조합을 `예정` 배지와 함께 시각화한다.
